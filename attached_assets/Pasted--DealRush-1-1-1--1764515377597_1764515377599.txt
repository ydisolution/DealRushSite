🎯 1. קונסספט ומהות הפלטפורמה
1.1 רעיון הליבה
DealRush היא פלטפורמת קניות קבוצתיות שבה המחיר יורד ככל שיותר אנשים קונים. זה לא קופון הנחה רגיל - זה מנגנון דינמי שבו:

משתמש ראשון משלם מחיר X
משתמש שני מצטרף → המחיר יורד ל-X-5%
משתמש שלישי מצטרף → המחיר יורד ל-X-7%
וכן הלאה עד הנחה מקסימלית (למשל 70%)
הפואנטה המרכזית: כולם משלמים את המחיר הנמוך ביותר שהושג עד סגירת הדיל. אם 200 אנשים הצטרפו והמחיר ירד ל-₪1,400, גם מי שהצטרף ראשון משלם ₪1,400.

7.1 מנגנון ה-Tier System
הגדרת Tiers למוצר:

javascript
// דוגמה למבנה נתונים
const deal = {
  productId: "AC-LG-12000",
  productName: "מזגן LG Inverter 12,000 BTU",
  originalPrice: 4500,
  currentPrice: 3690, // מתעדכן דינמית
  currentParticipants: 67,
  maxParticipants: 500, // ללא הגבלה באמת, רק לתצוגה
  endTime: "2024-11-18T14:00:00",
  
  tiers: [
    { min: 1, max: 30, discount: 10, price: 4050 },
    { min: 31, max: 60, discount: 15, price: 3825 },
    { min: 61, max: 100, discount: 18, price: 3690 },
    { min: 101, max: 150, discount: 22, price: 3510 },
    { min: 151, max: 200, discount: 30, price: 3150 },
    { min: 201, max: 999999, discount: 40, price: 2700 }
  ],
  
  participants: [
    { userId: "user123", joinedAt: "2024-11-15T10:30:00", lockedPrice: 3825 },
    { userId: "user456", joinedAt: "2024-11-15T11:45:00", lockedPrice: 3690 },

7.2 חישוב מחיר למשתמש
כשמשתמש מצטרף:

המחיר הנוכחי נעול עבורו (locked price)
זה המחיר המקסימלי שהוא ישלם
כשהדיל נסגר, כולם משלמים את finalPrice
כשהדיל נסגר:

javascript
const finalPrice = deal.currentPrice; // המחיר הנמוך ביותר שהושג

deal.participants.forEach(participant => {
  const amountToPay = finalPrice; // כולם משלמים אותו מחיר
  chargeUser(participant.userId, amountToPay);
  
  // אם המחיר שהמשתמש נעל היה גבוה יותר, הוא חסך
  const saved = participant.lockedPrice - finalPrice;
  sendNotification(participant.userId, `חסכת עוד ₪${saved}!`);
});
```

### **7.5 מניעת מניפולציות**

**בעיות אפשריות:**
1. משתמש מצטרף, רואה שהמחיר לא יורד, מבטל
2. משתמש ממתין לרגע האחרון
3. משתמש יוצר חשבונות פיקטיביים

**פתרונות:**
1. **No Cancellation**: לא ניתן לבטל לאחר הצטרפות (רק החזר אחרי קבלת המוצר)
2. **Locked Price**: המחיר נעול במקסימום, אז אין סיכון לחכות
3. **Verification**: אימות טלפון/אימייל, מניעת חשבונות כפולים
```
